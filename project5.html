Bark It! Interactive Cognitive Trainer

1. Introduction & Motivation

The Mission: "Bark It!" is an automated, sensor-driven gaming system designed to provide both mental and physical enrichment for canines. The project goal was to build a device that challenges dogs with varying stimuli, rewarding them for correct actions while providing entertainment for their owners through a competitive scoring system.

The Mechanics: Inspired by the classic "Bop It!" toy, the system randomly issues one of three commands—"Tug It," "Dig It," or "Place It." The dog must interpret the auditory cue and interact with the correct physical sensor within a decreasing time limit. Success extends the game streak and increases the speed, while failure triggers a game-over state.

My Contribution: Working in a multidisciplinary team, I focused heavily on the hardware integration and PCB design. I was responsible for selecting robust sensors capable of withstanding canine interaction, designing the custom circuit board to minimize wiring clutter, and validating the mixed-signal logic between the sensors and the microcontroller.

Preliminary Testing: Validating Sensor Logic and Game Loop

Sensor Architecture

The system relies on three distinct electromechanical inputs, each requiring unique signal processing:

  • Tug It (Analog): A rope tethered to an analog joystick. The system reads voltage changes on the X/Y axes to detect a "pull" event.
  • Dig It (Digital): A large, high-visibility red push button. The firmware implements a counter to detect three rapid presses, simulating a digging motion.
  • Place It (State Machine): A ball transfer task monitored by dual limit switches. The logic requires a specific sequence (Switch A releases -> Switch B triggers) to validate the move.
Hardware Specifications
  • Microcontroller: ATMega32 (8-bit AVR).
  • Display: 16x2 LCD for score tracking and visual feedback.
  • Audio: Active Buzzer for distinct command tones.
  • PCB: Custom 2-layer board with modular 2-pin connectors.
  • Enclosure: Laser-cut 1/8" plywood with sliding joinery.

3. Hardware Design & PCB Implementation

Transitioning from a breadboard mess to a reliable product was critical, especially given the physical nature of the user (a dog). I led the design of a custom PCB to centralize all connections.

PCB Design Strategy

I designed the PCB to serve as a modular "motherboard" for the doghouse enclosure.

  • Modular I/O: Instead of soldering sensors directly, I implemented JST and standard 0.1" headers for all external components (Joystick, Buttons, Switches). This allowed us to route wires through the enclosure walls easily during assembly.
  • Noise Filtering: The joystick lines included pull-down resistors and capacitors to prevent floating signals from triggering false "Tug" events.
  • Layout Optimization: The Keep-Out layer was utilized to ensure safety margins, and traces were routed to separate the noisy buzzer PWM signals from the sensitive analog joystick lines.
The "Place It" Engineering Challenge

Problem: Initially, we used an ultrasonic sensor to detect the ball. However, the dog's fur and erratic movement caused constant false positives, breaking the game loop.

Solution: We pivoted to a mechanical solution using **Limit Switches**. I designed a custom wooden lever arm inside the "Place It" bin. When the ball is dropped, its weight physically depresses the switch. This binary mechanical input proved 100% reliable compared to the noisy acoustic data of the ultrasonic sensor.

4. Firmware Logic (Sample Snippets)

The firmware operates on a tight loop that manages random command generation and input polling. Below is a representation of the input validation logic used for the "Dig It" command.

File: Input_Validation.c (Sample) Role: Sensor Debouncing & Logic 
// Function to validate 'Dig It' (3 Rapid Presses)
bool checkDigIt() {
    int pressCount = 0;
    unsigned long startTime = millis();

    while ((millis() - startTime) < gameTimer) {
        // Read Button State (Active Low)
        if (digitalRead(DIG_BTN) == LOW) {
            delay(50); // Debounce delay
            if (digitalRead(DIG_BTN) == LOW) {
                pressCount++;
                while(digitalRead(DIG_BTN) == LOW); // Wait for release
            }
        }
        
        // Success Condition
        if (pressCount >= 3) {
            lcd.print("Good Boy!");
            return true;
        }
    }
    return false; // Timeout
}
