Bark It!: Interactive Cognitive Trainer

Client: Junior Design Project

Location: Pittsburgh, PA

Year: 2025

Bark It! is an automated, sensor-driven system designed to provide physical and mental enrichment for canines. By utilizing an ATMega32 microcontroller, the device facilitates a real-time gaming loop that tracks performance and scales difficulty based on the user's success rate.

Design Constraints & Objectives

The development of the trainer was guided by specific functional requirements to ensure accessibility for the animal and reliability for the owner:

  • Behavioral Interfacing: Tactile sensors, such as joysticks and large push buttons, were selected to accommodate natural canine movements including tugging and paw-pressing.
  • Cognitive Feedback: The system incorporates multimodal cues, using a 16x2 LCD for visual tracking and an audio module for distinct command tones.
  • Safety-First Architecture: All electronic components and wiring are isolated within a hollow base underneath the enclosure to prevent accidental damage during play.
  • Adaptive Timing: The firmware features a dynamic timing variable that reduces the task completion window as the score increases, maintaining high engagement levels.

System Architecture

The system randomly cycles through three primary behavioral commands, each requiring a unique sensor interaction:

  • Tug It (Joystick Logic): Detects directional tension when a rope is pulled beyond a calibrated voltage threshold.
  • Dig It (Tactile Input): Monitors a high-visibility red push button for a sequence of three rapid presses to simulate digging behavior.
  • Place It (State-Machine Logic): Monitors the displacement of a ball between two compartments using dual limit switches.

Electronic Design Implementation

I designed a 2-layer PCB optimized for modularity within the enclosure. To streamline assembly, I implemented 2-pin connectors for all I/O instead of direct soldering. This design choice allowed for organized wire routing from the central board to sensors distributed throughout the doghouse structure. The layout includes a keep-out layer to ensure signal integrity and strategic grouping of connectors to minimize interior clutter.

Software Validation & Control Logic

The firmware manages random task generation and continuous sensor polling. A critical engineering challenge involved the "Place It" command logic. The code was refined to monitor specific state changes: the first switch must move from active to inactive (ball removed) before the second switch moves from inactive to active (ball placed). Furthermore, the system performs continuous monitoring; any incorrect action outside of the active command triggers an immediate game-over state.

Mechanical Fabrication

The enclosure was modeled in SolidWorks and constructed from laser-cut 1/8 inch plywood. The structure uses sliding joints reinforced with wood glue for high durability. To ensure sensor reliability, I fabricated custom wooden mounts to position the limit switches at the precise height and angle required for ball activation. The final assembly features a "porch" for paw-based tasks and a "storage" area for cognitive object movement.

Design Iterations & Future Work

During development, it was determined that ultrasonic sensors were prone to false positives, leading to a pivot to physical limit switches for superior accuracy. Future iterations will focus on integrating voice-recorded commands to replace simple tones and non-volatile memory to allow high scores to persist after power-cycling.
