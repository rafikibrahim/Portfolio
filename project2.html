[cite_start]I developed a modular simulation engine designed to model complex electrical grids and perform critical steady-state and contingency studies[cite: 473, 475]. [cite_start]The software utilizes Object-Oriented Programming (OOP) to represent physical assets and implements advanced numerical methods to solve non-linear power flow equations[cite: 478, 479, 594].

Core Simulation Capabilities

The engine automates the complex mathematical modeling required for utility-scale grid analysis:

    [cite_start]
  • Newton-Raphson Solver: Implements a full iterative solver to determine system voltages and power flows within a specified tolerance[cite: 594].
    • [cite_start]
  • Automated Matrix Construction: Programmatically builds the system Y-Bus matrix by aggregating primitive admittance data from transmission lines and transformers[cite: 574, 585].
    • [cite_start]
  • Sub-transient Fault Studies: Performs positive and negative sequence analysis to determine network response during faulted conditions[cite: 595, 599].
    • [cite_start]
  • System Normalization: Calculates per-unit (pu) values and base impedance automatically for any grid configuration[cite: 515, 517, 535].

Object-Oriented Grid Modeling

[cite_start]

The software is built on a library of physical classes, each encapsulating specific electrical properties[cite: 480]:

    [cite_start]
  • Network Nodes (Bus Class): Manages voltage magnitudes, angles, and node types such as Slack, PV, and PQ[cite: 538, 541, 544].
    • [cite_start]
  • Transmission Lines & Transformers: Calculates impedance ($Z$) and admittance ($Y$) based on physical attributes like length and $X/R$ ratios[cite: 499, 506, 520, 531, 534].
    • [cite_start]
  • Line Geometry Logic: Sophisticated modeling of GMD and GMR based on conductor coordinates and bundle spacing to determine electrical constants[cite: 482, 483, 490, 497].
    • [cite_start]
  • Generation & Load: Represents active grid components with specified real and reactive power injections or demands[cite: 547, 552, 553, 554, 560].

Numerical Methods & Implementation

[cite_start]

A central feature of the project is the custom implementation of the Newton-Raphson mathematical framework[cite: 594]:

    [cite_start]
  • Jacobian Formulation: Developed the logic to compute all four quadrants ($J_1$ through $J_4$) of the Jacobian matrix, enabling the linearization of power equations[cite: 588, 589, 590, 591].
    • [cite_start]
  • Convergence Logic: Implemented real-time calculation of power mismatches to monitor and update system states iteratively[cite: 592, 593].
    • [cite_start]
  • Fault Analysis: Utilizes Z-Bus matrix inversion to calculate post-fault bus voltages and fault current magnitudes[cite: 598, 599].

Data Management & Reporting

[cite_start]The tool leverages the Pandas library to manage and display high-dimensional data, such as polar-form Y-Bus and Jacobian matrices[cite: 569, 585]. [cite_start]This integration provides a readable, tabular interface for engineers to verify grid stability, line current limits, and total system power losses[cite: 594, 597].
