High Voltage Power System Diagnostics Simulator

Platform: Python (NumPy, Pandas)

Technical Focus: Power Flow Analysis & Fault Diagnostics

Repository: View on GitHub

I developed a high-performance simulation engine designed to model large-scale electrical grids and perform critical steady-state and fault diagnostics. This tool serves as a comprehensive guide for modeling power system components using Python, encapsulating complex physical attributes into a modular, object-oriented framework.

Core Simulation Capabilities

The engine automates the high-level mathematical modeling required for utility-grade grid analysis:

  • Newton-Raphson Power Flow Solver: An iterative numerical method implemented to solve the non-linear power flow equations, calculating voltage magnitudes and angles across the network.
  • Sub-transient Fault Analysis: Specialized routines for determining network responses during positive and negative sequence faults, essential for protection coordination.
  • Automated Matrix Construction: Programmatically generates the system Y-Bus matrix by aggregating primitive admittance data from individual lines and transformers.
  • Per-Unit (PU) Normalization: Automatically calculates base system impedance and converts physical values to per-unit parameters for consistent multi-voltage level analysis.

Object-Oriented Grid Architecture

The software is built on a robust library of classes representing essential power system elements:

  • Bus Class: Manages node indices and identifies bus types (Slack, PV, PQ) while tracking voltage magnitude and phase angles.
  • TransmissionLine & LineCode: Calculates line impedance (Z) and shunt admittance (Y) by processing physical line geometry, bundle spacing, and conductor resistance (Partridge).
  • XFMR (Transformer): Models high/low voltage transitions, percentage impedance, and X/R ratios to integrate voltage step-ups/downs into the system matrix.
  • Generator & Load: Represents active power injections and reactive demands, serving as the boundary conditions for the power flow solver.

Numerical Methods & Algorithm Logic

The core value of this simulator lies in the ground-up implementation of complex power system algorithms:

  • Jacobian Matrix Computation: I developed the logic to assemble the full Jacobian matrix from four distinct quadrants (J1, J2, J3, J4), which is the most mathematically intensive part of the Newton-Raphson process.
  • Power Mismatch Validation: Real-time calculation of real (P) and reactive (Q) power mismatches to monitor convergence and ensure the physical accuracy of the grid state.
  • Z-Bus Matrix Inversion: A critical routine for fault analysis that inverts the Y-Bus to obtain the Z-Bus, allowing for the determination of post-fault voltages and current injections.

Professional Data Processing

The simulator leverages the Pandas library to manage and display high-dimensional data. This provides a clean, tabular interface for engineers to analyze polar-form matrices and identify current ampacity violations or system power losses.

Considerations & Future Enhancements

While the current script provides an in-depth understanding of power system components, I am working toward several enhancements:

  • Integration of Zero-Sequence modeling for asymmetrical fault analysis (L-G, L-L-G).
  • Development of a Graphical User Interface (GUI) to replace the current console-based input system.
  • Scaling the solver for large-scale networks with over 100+ buses using sparse matrix optimization.
