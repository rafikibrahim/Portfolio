Retro Multiplayer Game Console - Capstone Senior Design Project

Client: ECE 1896: Senior Design Project

Location: Pittsburgh, PA

Year: 2025

This project involved creating a **dual-screen multiplayer game console** with advanced functionalities, utilizing **embedded systems** and custom **hardware design**. The system runs **classic games** like **Tetris**, **Pong**, and **Minesweeper**, interfacing through **Bluetooth controllers** and delivering high-quality video via **VGA output** to two independent screens. As the **Electrical Engineer**, I led the design, implementation, and integration of **PCBs**, **power systems**, and **communication protocols**, all while adhering to rigorous engineering constraints and testing standards.

GitHub Repository: You can view the complete project details, including code, schematics, and documentation, on my GitHub repository.

Design Constraints & Challenges

Before finalizing the design of the system, we conducted rigorous user testing and prototyping to address the key **design constraints**. Some of the main challenges encountered during development include:

  • Power Management: The system needed to ensure **72 hours** of continuous gameplay without recharging. This was achieved by utilizing **Li-ion batteries** and **DC-DC step-up converters**, carefully calculating the energy consumption for each component to avoid power failure during extended use.
  • Low-Latency Bluetooth Communication: The game required **real-time wireless communication** between the controllers and the console. Achieving low-latency Bluetooth communication while maintaining reliable data transmission was crucial, and optimizations were made to minimize input lag.
  • VGA Output Synchronization: Generating stable **VGA signals** with proper synchronization for **dual-screen output** was a critical challenge. The system used **DMA (Direct Memory Access)** on the **ESP32 microcontroller** to ensure smooth rendering of high-resolution video.
  • Hardware Integration: Integrating various subsystems, such as game logic, controller inputs, and video output, required careful planning and signal routing. The PCB design process had to balance complexity with efficiency to keep the system compact and functional.

System Overview & Design Features

The system architecture was carefully designed to ensure smooth interaction between hardware components. Key features of the system include:

  • ESP32 Microcontrollers: At the heart of the console are two **ESP32 microcontrollers** that manage the game logic, video output, and communication between the controllers and the main console.
  • Bluetooth Communication: The controllers communicate wirelessly with the console using **HC-05 Bluetooth modules**, ensuring a **low-latency connection** between the player inputs and game state updates.
  • VGA Output: The **ESP32** generates **VGA video signals** to drive the two independent screens. **DMA techniques** are used to manage the large amount of data required for smooth video rendering.
  • Custom Controller Design: The **controller PCB** integrates an **ATmega328P microcontroller** to process input from the joystick and buttons, communicating with the console over Bluetooth. It is powered by a **Li-ion rechargeable battery** and designed for ergonomic comfort.

Future Improvements

While the system is fully functional, several areas could be improved for future versions:

  • Extended Game Library: Expanding the range of available games and introducing more complex multiplayer features could improve user engagement and longevity of the system.
  • Controller Design Refinement: Refining the **ergonomics** of the controller to make it more comfortable for extended use and adding **rumble feedback** to increase the immersion of gameplay.
  • Enclosure Design: Creating a **compact and durable enclosure** to house both the console and controllers, making the system more portable and user-friendly for on-the-go gaming.
