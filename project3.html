Retro Multiplayer Game Console - Capstone Senior Design Project

Client: ECE 1896: Senior Design Project

Location: Pittsburgh, PA

Year: 2025

This project involved creating a **dual-screen multiplayer game console** capable of running **classic games** like **Tetris**, **Pong**, and **Minesweeper**. The system was designed to interface with Bluetooth controllers and output via VGA to two independent screens. As the **Electrical Engineer**, I led the design of the hardware components, including **PCB development**, **Bluetooth communication**, and **power management** to ensure a seamless gaming experience.

GitHub Repository: You can view the complete project details, including code, schematics, and documentation, on my GitHub repository.

Design Constraints & Challenges

During the design process, we faced several constraints and challenges that had to be addressed:

  • Power Management: The system needed to run for **72 hours** of continuous gameplay, requiring efficient battery management and low power consumption.
  • Bluetooth Latency: Ensuring **low-latency communication** between the controllers and console for real-time interaction was a key challenge.
  • Hardware Integration: Integrating the **VGA video output** and ensuring synchronization between multiple subsystems (game logic, controller input, and video output) was complex.
  • Multiplayer Synchronization: Achieving smooth synchronization between the two players' actions and game states was crucial for the gaming experience.

Technical Design

The system is based on two **ESP32** microcontrollers, which are responsible for managing game logic, video generation, and communication between the controllers. The main design features are as follows:

  • Power System: The controllers use **Li-ion batteries**, and the system is powered by a **DC-DC step-up converter** to provide the necessary 5V output.
  • VGA Output: **VGA signals** are generated by the **ESP32** using **DMA** to ensure smooth video output on two separate screens.
  • Bluetooth Communication: The **HC-05 Bluetooth module** is used for wireless communication between the controllers and the console, with low-latency response times for real-time control.
  • Controller Design: The **controller PCB** integrates the joystick and buttons and communicates with the console via Bluetooth.

Future Improvements

Looking forward, there are several improvements that could be made to enhance the project:

  • Extended Game Library: Adding more games and improving multiplayer features.
  • Improved Controller Design: Refining the controller for better ergonomics and adding more advanced features like **rumble feedback**.
  • Physical Enclosure: Designing a more compact and durable **enclosure** for the console and controllers.
