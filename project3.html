Retro Multiplayer Game Console - Capstone Senior Design Project

Client: ECE 1896: Senior Design Project

Location: Pittsburgh, PA

Year: 2025

This project involved the creation of a **dual-screen multiplayer game console** with advanced functionalities, utilizing **embedded systems** and custom **hardware design**. The system runs **classic games** like **Tetris**, **Pong**, and **Minesweeper**, interfacing through **Bluetooth controllers** and delivering high-quality video via **VGA output** to two independent screens. As the **Electrical Engineer**, I spearheaded the design, implementation, and integration of **PCBs**, **power systems**, and **communication protocols**, all while adhering to rigorous engineering constraints and testing standards.

GitHub Repository: You can view the complete project details, including code, schematics, and documentation, on my GitHub repository.

Design Constraints & Challenges

The project presented several **engineering challenges** that required innovative solutions. Here are the most significant constraints:

  • Power Management: One of the most demanding aspects was ensuring that the system could operate continuously for **72 hours** of gameplay without recharging. This was achieved by utilizing **Li-ion batteries** and integrating **DC-DC step-up converters** for power regulation.
  • Low-Latency Bluetooth Communication: Achieving **low-latency wireless communication** between the controllers and console was critical. Bluetooth communication was optimized for **real-time interaction** with minimal input lag, ensuring a seamless multiplayer experience.
  • VGA Video Synchronization: Generating **VGA signals** required precise **timing control** to ensure the video output remained synchronized across both screens. This required advanced programming techniques and careful selection of the **ESP32 microcontroller** for video output generation.
  • Hardware Integration: The integration of different subsystems (game logic, controller input, video output, and Bluetooth communication) was complex. Careful **PCB design**, signal routing, and timing analysis were necessary to ensure smooth communication between components.

Technical Design

The system utilizes two **ESP32 microcontrollers** that communicate over **UART** for **multiplayer synchronization** and generate the **VGA video output**. Key technical features include:

  • Power System: The system operates using **Li-ion rechargeable batteries** and a **DC-DC step-up converter** to ensure that both the main console and the controllers maintain stable operation for long durations.
  • Bluetooth Communication: **HC-05 Bluetooth modules** are employed to facilitate low-latency communication between the controllers and the main console. This was critical for real-time input and output synchronization.
  • VGA Output Generation: **VGA signals** are generated using the **ESP32 microcontroller**, which drives two screens with precise timing and pixel data. This was achieved using **DMA (Direct Memory Access)** techniques for efficient video rendering.
  • Controller Design: The **controller PCB** integrates **ATmega328P** microcontrollers with **joystick** and **button** inputs, which communicate wirelessly via Bluetooth to the main console.

Future Improvements

Moving forward, there are several key improvements that could further enhance the project:

  • Extended Game Library: Adding more games to the library and introducing more complex **multiplayer modes** to improve gameplay experience.
