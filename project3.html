Retro Multiplayer Game Console - Capstone Senior Design Project

Client: ECE 1896: Senior Design Project

Location: Pittsburgh, PA

Year: 2025

This project involved creating a **dual-screen multiplayer game console** that combines **embedded systems**, **hardware design**, and **communication protocols**. The console is capable of running **classic games** like **Tetris**, **Pong**, and **Minesweeper**, and it interfaces via **Bluetooth controllers** with **VGA output** to drive two independent screens. The project was highly complex due to the need for **real-time interaction** between the console and controllers, along with the precise synchronization of **dual-screen output**. As the **Electrical Engineer**, I led the design and integration of the **PCB**, **power systems**, **VGA video generation**, and **Bluetooth communication**, addressing the numerous engineering challenges along the way.

GitHub Repository: All project files, including code, schematics, and detailed documentation, are available on my GitHub repository.

Design Constraints & Challenges

Throughout the development of this project, several engineering challenges and design constraints were identified and addressed, such as:

  • Power Efficiency: Ensuring the system could operate for **72 hours** of continuous gameplay on **Li-ion rechargeable batteries** without requiring recharging. This involved designing **DC-DC step-up converters** and optimizing power consumption for each component.
  • Low-Latency Bluetooth Communication: Optimizing **Bluetooth communication** to minimize **input lag** and ensure that the controllers communicated **real-time data** to the console without any noticeable delay, which is critical for gaming.
  • Synchronization of Dual-Screen Output: Implementing **precise VGA timing** to drive **two independent VGA displays** for multiplayer functionality, which required efficient use of **Direct Memory Access (DMA)** techniques on the **ESP32 microcontroller** to handle video rendering.
  • Hardware Integration: The integration of multiple subsystems (game logic, Bluetooth input, and VGA output) required careful **signal routing** on the PCB and ensuring the components did not interfere with one another.

System Overview & Design Features

The architecture consists of several key components that work in harmony to deliver a seamless multiplayer experience:

  • Dual ESP32 Microcontrollers: The two **ESP32 microcontrollers** handle **game logic**, **Bluetooth communication**, and **VGA signal generation** for both players.
  • Bluetooth Communication: **HC-05 Bluetooth modules** provide wireless communication between the controllers and console, ensuring near-instantaneous interaction.
  • VGA Output: **VGA output** is generated by the **ESP32 microcontroller**, which uses **DMA** to efficiently handle the large data flow required for smooth **dual-screen rendering**.
  • Controller Design: The custom **controller PCB** integrates an **ATmega328P microcontroller**, joystick, buttons, and Bluetooth communication to send player input to the console.

Technical Design & Features

The technical design involves using the **ESP32 microcontroller** to run the core logic, handle real-time communication with controllers, and generate VGA signals for display. Key features include:

  • Real-Time Gameplay: Each game (Tetris, Pong, Minesweeper) runs on the **ESP32 microcontroller**, which processes the game logic and updates the screen in real-time.
  • Multiplayer Synchronization: The **two ESP32 microcontrollers** communicate with each other via **UART** to ensure smooth synchronization between the players.
  • Power Management: Efficient use of **DC-DC converters** and **Li-ion batteries** ensures the system can operate for extended periods without recharging.
  • Audio & Visual Subsystems: The audio subsystem offloads sound processing to an **ATmega328P microcontroller**, while the **VGA output** is handled using DMA to ensure high-quality video rendering.
