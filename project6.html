Autonomous Line-Following Robot

Platform: Arduino (C++/Embedded C)

Technical Focus: Real-time Control Systems, PWM Optimization, and Infrared Sensing

I designed and implemented an autonomous mobile robotics system engineered to track high-contrast paths using a dual-sensor array. The project involved solving critical hardware-software integration challenges, specifically regarding motor torque at low speeds and sensor calibration under varying ambient light conditions.

Project Demonstration

1. System Architecture & Hardware

The robot is built on a modular chassis designed for stability and rapid response:

  • Central Processing Unit: An Arduino Uno executes the navigation algorithms and manages high-frequency PWM signals.
  • Sensing Array: Dual IR (Infrared) sensor modules are mounted at the front of the chassis. These sensors detect the black line by measuring the reflection of infrared light off the surface.
  • Motor Control: Integrated an L298N H-Bridge motor driver to facilitate independent bi-directional control of the drive wheels.
  • Drive Train: High-torque TT Gear Motors provide the necessary actuation for precise steering and consistent forward velocity.
  • Power Management: Utilized a 7.4V LiPo battery system to provide a high discharge rate required for consistent motor performance.

2. Control Logic & Navigation Algorithm

The robot operates on a closed-loop feedback system that continuously polls the IR sensors to adjust its trajectory:

  • Forward State: When neither sensor detects the black line (both sensors on the white surface), the controller maintains equal PWM signals to both motors for straight-line motion.
  • Differential Steering:
    • Right Correction: If the right sensor detects the black line, the algorithm reverses the right motor while moving the left motor forward to pivot the robot back onto the path.
    • Left Correction: Conversely, if the left sensor detects the line, the left motor reverses to correct the heading.
  • Stop State: If both sensors detect the black line (e.g., at a finish line), the system executes a full stop command to all motor pins.

3. Engineering Challenges & Optimization

A major technical hurdle in this project was the non-linear behavior of the gear motors at low speeds. I implemented several optimizations to ensure smooth movement:

  • High-Frequency PWM: Standard PWM frequencies often cause TT motors to stall or "hum" at low duty cycles. I optimized the code to increase the PWM frequency to approximately 7.8 kHz. This ensures the motors run smoothly at low speeds with sufficient torque to overcome static friction.
  • Sensitivity Calibration: I implemented a hardware-level calibration routine using the onboard potentiometers of the IR modules to ensure a 4cm wide path could be reliably detected regardless of the floor's reflectivity.
  • Adaptive Speed Scaling: The software includes a variable speed constant that allows for real-time calibration based on battery voltage drop, maintaining a consistent performance profile throughout the discharge cycle.

4. Performance Results

The final implementation successfully tracked complex curves and 90-degree turns with minimal oscillation. By fine-tuning the differential steering delays, I achieved a balanced ratio between speed and tracking accuracy, making the system robust against minor path imperfections.
