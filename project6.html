Autonomous Line-Following Robot

1. Introduction & Objectives

The Goal: To design and engineer an autonomous mobile robot capable of tracking a high-contrast path (black line on white surface) with high precision. This project focuses on closed-loop control systems, sensor fusion, and actuator management using a compact embedded platform.

The Solution: I engineered a differential-drive robot powered by an Arduino Nano. The system utilizes active Infrared (IR) Reflectance sensors to constantly poll the ground surface. By processing these binary signals in real-time, the microcontroller adjusts the RPM of two independent DC motors via an L293D driver, allowing the robot to perform course corrections (differential steering) instantly when it deviates from the trajectory.

System Validation: Sensor Calibration & Track Testing

Core Hardware Architecture

Unlike standard kits, I optimized this build for compactness and higher torque by upgrading the power system and processing unit.

  • Controller: Arduino Nano (ATmega328P) - chosen for its small footprint compared to the Uno.
  • Actuation: L293D H-Bridge Motor Driver Module for bidirectional motor control.
  • Sensing: Dual IR Proximity Sensors (LM393 Comparator based) with adjustable sensitivity.
  • Power Plant: 8x AA Batteries (Series Config) delivering ~12VDC for maximum motor torque.
Power System Engineering

Standard builds often use 4 AA batteries (6V), which causes voltage sag under load. I engineered a 12V Power Rail using 8 AA batteries.

  • Motor Isolation: The 12V rail drives the motors directly, providing high torque for sharp turns.
  • Logic Regulation: The Arduino Nano's onboard VIN regulator steps the 12V down to a stable 5V for the logic circuits and sensors, ensuring processor stability even when motors stall.

2. Algorithmic Logic & Control Strategy

The robot operates on a decision-tree algorithm based on binary sensor inputs. This allows for immediate reaction to track changes.

Scenario Left Sensor Right Sensor Control Action (Differential Drive) Result
Straight Path Low (White) Low (White) Left Motor: FWD | Right Motor: FWD Robot moves forward at full speed.
Right Turn Low (White) High (Black) Left Motor: FWD | Right Motor: STOP/REV Robot pivots Right to realign.
Left Turn High (Black) Low (White) Left Motor: STOP/REV | Right Motor: FWD Robot pivots Left to realign.
Stop / Intersection High (Black) High (Black) Left Motor: STOP | Right Motor: STOP Robot halts (End of Track).

3. Firmware Implementation (C++)

The firmware was written in C++ for the Arduino environment. It utilizes direct port manipulation for motor control to minimize latency. Below is the core logic loop implementing the control table above.

File: LineFollower_Nano.ino Role: Main Control Loop 
void loop() {
  // 1. Read Sensor Data
  int leftValue = digitalRead(LEFT_SENSOR_PIN);
  int rightValue = digitalRead(RIGHT_SENSOR_PIN);

  // 2. Decision Logic (Active High Sensors)
  if (leftValue == LOW && rightValue == LOW) {
    // CASE 1: Forward
    motorLeft.run(FORWARD);
    motorRight.run(FORWARD);
  }
  else if (leftValue == HIGH && rightValue == LOW) {
    // CASE 2: Turn Left (Pivot)
    motorLeft.run(BACKWARD); // Reverse for sharper turn
    motorRight.run(FORWARD);
  }
  else if (leftValue == LOW && rightValue == HIGH) {
    // CASE 3: Turn Right (Pivot)
    motorLeft.run(FORWARD);
    motorRight.run(BACKWARD);
  }
  else {
    // CASE 4: Stop
    motorLeft.run(RELEASE);
    motorRight.run(RELEASE);
  }
}
