RFID Smart Car Gate Opener

1. Introduction & Project Scope

[cite_start]Concept & Inspiration: This project is a functional prototype of an automated parking gate system [cite: 1340][cite_start], directly inspired by the gate system at the Soldiers and Sailors Garage where I park daily for university[cite: 1352]. Fascinated by the mechanism that controls secure vehicle entry, I challenged myself to replicate and improve upon this real-world system using embedded technology.

The Solution: The system addresses the need for secure, efficient, and contactless vehicle management. [cite_start]Unlike simple manual gates, this prototype integrates an RFID-based authentication system with an automated mechanical gate[cite: 1340]. [cite_start]It features an interactive 16x2 LCD interface to prompt users [cite: 1341][cite_start], a servo-actuated gate arm [cite: 1342][cite_start], and an intelligent ultrasonic sensor loop that prevents the gate from closing until the vehicle has safely passed[cite: 1344].

My Role : As an individual project, I was responsible for every phase of the engineering lifecycle:

    [cite_start]
  • Electrical Engineering: Designed the mixed-voltage power architecture and fabricated the custom PCB[cite: 2027].
    • [cite_start]
  • Firmware Engineering: Wrote the C++ firmware to coordinate the RFID reader, servo timing, and safety sensors[cite: 1367].
    • [cite_start]
  • Mechanical Design: Modeled and fabricated a custom enclosure using SolidWorks and woodworking tools[cite: 2150].
System Architecture

[cite_start]The system is built around a standalone ATmega328P microcontroller[cite: 1367]. It features a dual-voltage power architecture to support mixed-signal components.

  • Processing: ATmega328P (16MHz Internal Clock).
    • [cite_start]
  • Authentication: MFRC522 RFID Module (13.56 MHz)[cite: 1365].
    • [cite_start]
  • User Interface: 16x2 LCD (I2C) + Piezo Speaker + Status LEDs[cite: 1363, 1371].
    • [cite_start]
  • Actuation: Micro Servo Motor (PWM Controlled)[cite: 1364].
    • [cite_start]
  • Safety: HC-SR04 Ultrasonic Sensor for vehicle detection[cite: 1366].
Design Challenges Solved
    [cite_start]
  • Mixed Voltage Logic: The RFID sensor requires 3.3V logic, while the servo and LCD require 5V[cite: 1777]. [cite_start]I designed a custom regulator circuit using LM7805 and LM2937 chips to bridge these domains safely[cite: 2018].
  • Resource Optimization: With limited pins on the ATmega328P, I utilized the I2C protocol for the LCD to save GPIO lines for sensors.
    • [cite_start]
  • Power Stability: To prevent servo motor noise from resetting the microcontroller, I implemented decoupling capacitors and separate power rails on the PCB[cite: 2031].

3. Hardware Design & PCB Implementation

Moving from a breadboard prototype to a final product required a custom PCB design. [cite_start]The primary challenge was integrating the voltage regulation directly onto the board to eliminate external power bricks[cite: 2017].

Power Architecture

The system is powered by a 9V source. I engineered a dual-regulator circuit directly on the PCB:

    [cite_start]
  • 5V Rail: An LM7805CV linear regulator powers the ATmega328P, LCD, and Servo[cite: 2018].
    • [cite_start]
  • 3.3V Rail: An LM2937ET-3.3 regulator provides clean power to the sensitive MFRC522 RFID module[cite: 2031].
    • [cite_start]
  • Thermal Management: Regulators were spaced to dissipate heat, and decoupling capacitors were placed close to the power pins to filter high-frequency noise[cite: 2032].
PCB Design Strategy

[cite_start]Designed in Altium Designer[cite: 2030], the PCB features:

    [cite_start]
  • Modular Headers: Instead of soldering components directly, I used female headers for the RFID, Ultrasonic, and Servo connections[cite: 2033]. This allows for easy replacement of faulty sensors.
    • [cite_start]
  • ISP Programming Header: Included to allow in-circuit programming of the ATmega328P without removing the chip[cite: 2035].
    • [cite_start]
  • Compact Layout: Traces were routed to minimize cross-talk between the high-current servo lines and the sensitive RFID SPI lines[cite: 2037].

4. Firmware & Software Implementation (Sample Code)

I wrote the firmware in C++ using the Arduino IDE. The logic relies on a state machine that handles card scanning, authentication, and safety checks before moving the gate. [cite_start]Below are sample snippets of the core logic I implemented[cite: 1431].

File: RFID_Logic.ino (Sample) Role: Authentication & Access Control 
void loop() {
    // 1. Check for new card presence
    if (!mfrc522.PICC_IsNewCardPresent()) return;
    if (!mfrc522.PICC_ReadCardSerial()) return;

    // 2. Read UID and Compare with Authorized Database
    String content = "";
    for (byte i = 0; i < mfrc522.uid.size; i++) {
        content.concat(String(mfrc522.uid.uidByte[i] < 0x10 ? " 0" : " "));
        content.concat(String(mfrc522.uid.uidByte[i], HEX));
    }
    content.toUpperCase();

    // 3. Determine Access State
    if (content.substring(1) == AUTHORIZED_UID) {
        grantAccess(); // Trigger Green LED, Servo Open, Success Tone
    } else {
        denyAccess();  // Trigger Red LED, Error Tone
    }
}
File: Gate_Control.ino (Sample) Role: Actuation & Safety Sensing 
void grantAccess() {
    lcd.clear();
    lcd.print("Access Granted");
    
    // 1. Open Gate
    servo.write(90); 
    delay(2000); 

    // 2. Safety Loop: Wait for vehicle to pass using Ultrasonic Sensor
    while (isObjectDetected()) {
        delay(100); // Hold gate open if car is detected
    }

    // 3. Close Gate after vehicle clears
    delay(1000); 
    servo.write(0); 
    resetSystem();
}

bool isObjectDetected() {
    // Ultrasonic Logic: Trigger Pulse -> Echo Read
    digitalWrite(TRIG_PIN, LOW);
    delayMicroseconds(2);
    digitalWrite(TRIG_PIN, HIGH);
    delayMicroseconds(10);
    digitalWrite(TRIG_PIN, LOW);
    
    long duration = pulseIn(ECHO_PIN, HIGH);
    int distance = duration * 0.034 / 2;
    
    return (distance < 10); // Return TRUE if object is within 10cm
}
