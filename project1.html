Autonomous Line-Following Robot

1. Executive Summary & Control Strategy

Project Scope: The objective was to engineer a high-performance autonomous mobile robot capable of tracking high-contrast trajectories with minimal latency. This project is not merely a hobbyist build; it is a rigorous exercise in Systems Engineering, integrating mechanical dynamics, power electronics, and embedded control theory into a cohesive unit. The system was designed to simulate the guidance logic used in industrial Automated Guided Vehicles (AGVs) within warehouse logistics.

Architecture: The robot is built upon the Arduino Nano (ATmega328P) architecture, selected for its optimal balance of GPIO capability and physical footprint. The control loop operates on a Sensor Fusion strategy, aggregating data from active IR reflectance modules to make microsecond-level decisions. This data drives a custom differential steering algorithm that manipulates a Dual H-Bridge (L293D) to execute rapid course corrections.

Power Engineering: A significant portion of the design phase was dedicated to power integrity. Standard 6V robotic kits frequently suffer from voltage sag during high-torque startup events, leading to microcontroller brownouts. To mitigate this, I engineered a robust 12V DC Rail using 8x AA batteries in a series configuration. This high-voltage domain drives the motors directly to overcome static friction, while the Arduino's onboard voltage regulator provides a clean, isolated 5V logic rail for the processor and sensors.

Core Hardware Specifications

The bill of materials was optimized to ensure high torque-to-weight ratios and efficient thermal management under continuous load.

  • Processing Unit: Arduino Nano V3 (16MHz, ATmega328P) using bare-metal optimization.
  • Motor Driver: L293D Dual H-Bridge with internal flyback diodes for back-EMF protection.
  • Sensing Array: Dual Active IR Reflectance Modules with LM393 Comparators.
  • Chassis Dynamics: 2WD Differential Drive with caster stabilization.
  • Actuators: 2x Geared DC Motors (High Torque, 1:48 Gear Ratio).
Low-Level Firmware Optimization

To achieve high-fidelity motor control on an 8-bit microcontroller, standard libraries were bypassed in favor of direct register access.

  • Register Manipulation: Direct modification of the TCCR0B register to alter the timer prescaler. This shifted the PWM frequency from the standard 490Hz to approx. 7.8kHz, significantly reducing audible motor whine and smoothing the torque curve at low speeds.
  • Bitwise Logic: Utilized bitwise shift operations to compress sensor states into a single integer, allowing for O(1) decision complexity in the main control loop.

2. Control Algorithm & State Machine

The robot utilizes a Bang-Bang Control System (a subset of hysteresis control) for its decision logic. While a PID controller offers smoother transitions, the Bang-Bang approach provides superior reaction times for sharp 90-degree turns often found in competitive line-following tracks. The logic is implemented as a high-frequency polling loop.

State Vector (Binary) Left Sensor Right Sensor Control Action (Differential Drive) System Behavior
0b00 (Track Lock) Low (White) Low (White) Left: FWD | Right: FWD Maintain maximal linear velocity.
0b01 (Left Drift) Low (White) High (Black) Left: FWD | Right: REV Counter-Torque: Pivot Right to regain track lock.
0b10 (Right Drift) High (Black) Low (White) Left: REV | Right: FWD Counter-Torque: Pivot Left to regain track lock.
0b11 (Intersection) High (Black) High (Black) Left: STOP | Right: STOP End of Track / Intersection Safety Stop.

3. Complete Firmware Source Code (C++)

Below is the complete, modular C++ source code flashed to the microcontroller. The code is structured with function prototypes for maintainability and utilizes direct hardware abstraction for efficiency. Note the specific inclusion of the setPWMFrequency() function, which manipulates the ATmega328P hardware timers.

File: Autonomous_Nav_Main.ino Role: Production Firmware 
// ---------------------------------------------------------
// AUTONOMOUS LINE FOLLOWER FIRMWARE
// Author: Rafik Ibrahim
// Platform: Arduino Nano (ATmega328P)
// ---------------------------------------------------------

// Define pins for the IR sensors (Active High Logic)
#define IR_SENSOR_RIGHT 11 // Digital Input for Right Sensor
#define IR_SENSOR_LEFT 12  // Digital Input for Left Sensor

// Define the motor speed (PWM Duty Cycle: 0-255)
#define MOTOR_SPEED 180 

// Define pins for the RIGHT motor (H-Bridge Channel A)
const int enableRightMotor = 6; // PWM Enable Pin
const int rightMotorPin1 = 7;   // Logic Input 1 (Direction)
const int rightMotorPin2 = 8;   // Logic Input 2 (Direction)

// Define pins for the LEFT motor (H-Bridge Channel B)
const int enableLeftMotor = 5;  // PWM Enable Pin
const int leftMotorPin1 = 9;    // Logic Input 3 (Direction)
const int leftMotorPin2 = 10;   // Logic Input 4 (Direction)

// Function Prototypes for Modular Architecture
void setupMotors();         // Hardware abstraction initialization
void setPWMFrequency();     // Direct register manipulation for PWM
void controlMotors(int leftSpeed, int rightSpeed); // Differential drive logic
int readLineSensors();      // Sensor fusion and state determination

void setup()
{
  // 1. Initialize GPIO pins for H-Bridge control
  setupMotors();

  // 2. Optimize PWM frequency to ~7.8kHz to reduce motor noise
  setPWMFrequency();

  // 3. Initialize IR Sensor Inputs
  pinMode(IR_SENSOR_RIGHT, INPUT);
  pinMode(IR_SENSOR_LEFT, INPUT);

  // 4. Safety State: Ensure motors are stopped on boot
  controlMotors(0, 0);
}

void loop()
{
  // Real-time Polling Loop
  
  // 1. Acquire current sensor state vector
  int sensorState = readLineSensors();

  // 2. Execute Control Action based on State Vector
  switch (sensorState)
  {
    case 0b00: // State: No line detected (Centered on white)
      controlMotors(MOTOR_SPEED, MOTOR_SPEED); // Action: Forward
      break;

    case 0b01: // State: Left sensor triggered
      // Differential Steering: Left Motor FWD / Right Motor REV
      controlMotors(MOTOR_SPEED, -MOTOR_SPEED); 
      break;

    case 0b10: // State: Right sensor triggered
      // Differential Steering: Left Motor REV / Right Motor FWD
      controlMotors(-MOTOR_SPEED, MOTOR_SPEED); 
      break;

    case 0b11: // State: Both sensors triggered (Intersection/End)
      controlMotors(0, 0); // Action: Hard Stop
      break;

    default: // Failsafe state
      controlMotors(0, 0);
  }
}

// Hardware Abstraction Layer: Motor Initialization
void setupMotors()
{
  pinMode(enableRightMotor, OUTPUT);
  pinMode(rightMotorPin1, OUTPUT);
  pinMode(rightMotorPin2, OUTPUT);

  pinMode(enableLeftMotor, OUTPUT);
  pinMode(leftMotorPin1, OUTPUT);
  pinMode(leftMotorPin2, OUTPUT);
}

// Advanced Optimization: Direct Timer Register Access
void setPWMFrequency()
{
  // Modifying Timer0 prescaler. Default is 0x03 (64 prescaler).
  // Setting to 0x02 (8 prescaler) increases freq from 490Hz to ~7.8kHz.
  // This moves the PWM frequency out of the audible resonance range of the motor coils.
  TCCR0B = TCCR0B & 0b11111000 | 0b00000010;
}

// Differential Drive Controller
void controlMotors(int rightSpeed, int leftSpeed)
{
  // Right Motor H-Bridge Logic
  digitalWrite(rightMotorPin1, rightSpeed > 0); // TRUE if FWD
  digitalWrite(rightMotorPin2, rightSpeed < 0); // TRUE if REV

  // Left Motor H-Bridge Logic
  digitalWrite(leftMotorPin1, leftSpeed > 0); 
  digitalWrite(leftMotorPin2, leftSpeed < 0); 

  // Speed Injection via PWM (Absolute value handles reverse logic)
  analogWrite(enableRightMotor, abs(rightSpeed)); 
  analogWrite(enableLeftMotor, abs(leftSpeed));   
}

// Sensor Fusion Driver
int readLineSensors()
{
  // Acquire raw digital signals
  int rightSensor = digitalRead(IR_SENSOR_RIGHT); 
  int leftSensor = digitalRead(IR_SENSOR_LEFT);   

  // Compress states into a 2-bit integer for efficient switching
  // Returns: 0, 1, 2, or 3
  return (rightSensor << 1) | leftSensor;
}
